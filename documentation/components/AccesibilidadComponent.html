<main>
  <div class="toCenter">
    <h1>Accesibilidad</h1>

    <section>
      <h2>Etiquetas Alt y Title en las Imágenes</h2>
      <p>
        Las imágenes en el sitio web tienen atributos alt y/o title que
        describen su contenido. Esto es fundamental para usuarios con
        discapacidad visual que utilizan lectores de pantalla, ya que les
        permite comprender el significado de las imágenes.
      </p>
    </section>

    <section>
      <h2>Contraste de Color</h2>
      <p>
        Se ha aplicado un contraste adecuado entre el texto y el fondo para
        mejorar la legibilidad. Es importante que haya suficiente contraste
        entre el texto y el fondo para que sea fácil de leer, especialmente para
        usuarios con discapacidad visual o dificultades de visión.
      </p>
      <p>
        Se recomienda utilizar combinaciones de colores que cumplan con las
        pautas de accesibilidad, como el estándar WCAG (Web Content
        Accessibility Guidelines).
      </p>
    </section>

    <section>
      <h2>Estructura Lógica del Contenido</h2>
      <p>
        El contenido del sitio web está organizado de manera lógica y
        estructurada utilizando encabezados, listas y otros elementos HTML
        semánticos. Esto ayuda a los usuarios, especialmente a aquellos que
        utilizan tecnologías de asistencia como lectores de pantalla, a navegar
        por el contenido de manera más eficiente.
      </p>
      <p>
        Es importante utilizar los elementos HTML de acuerdo con su significado
        semántico. Por ejemplo, utilizar los elementos
        <code>&lt;header&gt;</code>, <code>&lt;main&gt;</code>,
        <code>&lt;footer&gt;</code>, <code>&lt;nav&gt;</code>, etc., de manera
        apropiada para estructurar el contenido de la página.
      </p>
    </section>

    <section>
      <h2>Web Responsive</h2>
      <p>
        Se ha tenido en cuenta el tamaño de los distintos dispositivos para
        ajustar el contenido al mismo. El diseño responsivo es fundamental para
        garantizar una experiencia de usuario óptima en una amplia gama de
        dispositivos y tamaños de pantalla, incluyendo computadoras de
        escritorio, tablets y teléfonos móviles.
      </p>
      <p>
        Al diseñar un sitio web responsivo, se debe considerar la disposición de
        los elementos, el tamaño de la fuente, la legibilidad y la usabilidad en
        diferentes dispositivos. Se pueden utilizar técnicas como el diseño
        basado en rejillas (grid layout) y los medios consultas (media queries)
        para lograr un diseño adaptable.
      </p>
    </section>

    <section>
      <h2>Uso de ARIA Roles y Atributos</h2>
      <p>
        Se han utilizado roles y atributos ARIA (Accessible Rich Internet
        Applications) para mejorar la accesibilidad de elementos interactivos y
        widgets personalizados. ARIA proporciona una manera de mejorar la
        accesibilidad de elementos HTML que no tienen una semántica adecuada
        para ciertos contextos.
      </p>
      <p>
        Es importante utilizar roles y atributos ARIA de manera adecuada y
        significativa para garantizar que los elementos sean accesibles para
        todos los usuarios, incluyendo aquellos que utilizan tecnologías de
        asistencia.
      </p>
    </section>

    <section>
      <h2>Navegación y Teclado</h2>
      <p>
        La navegación y la interacción con el sitio web pueden realizarse
        fácilmente utilizando solo el teclado. Esto es fundamental para usuarios
        con discapacidad motora o aquellos que prefieren no utilizar un mouse.
      </p>
      <p>
        Se han proporcionado atajos de teclado y se han implementado prácticas
        de accesibilidad como el enfoque visible (focus visible) para indicar
        claramente qué elemento está activo en la interfaz.
      </p>
    </section>

    <section>
      <h2>Pruebas de Accesibilidad</h2>
      <p>
        Se han realizado pruebas de accesibilidad utilizando herramientas
        automáticas y pruebas manuales para identificar y corregir posibles
        barreras de accesibilidad. Es importante realizar pruebas regulares para
        garantizar que el sitio web sea accesible para todos los usuarios.
      </p>
      <p>
        Las herramientas de evaluación de accesibilidad pueden proporcionar
        informes detallados sobre problemas de accesibilidad y sugerencias para
        mejorar la accesibilidad del sitio web.
      </p>
    </section>
  </div>
</main>
